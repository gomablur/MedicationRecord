# ロードマップ / 将来対応メモ

基本機能 (記録・確認・QR 取り込み・OAuth ログイン) 以外の要望と、その実現可能性の調査メモ。

## マイナポータル連携 (処方情報の自動取得)

**現状: 個人開発アプリからは事実上不可。手動連携の口だけ用意してある。**

- マイナポータルの「わたしの情報」API (薬剤情報・診療情報) は、デジタル庁の
  **API 利用申請を通過した事業者**にのみ開放される (医療系サービス事業者向け。審査あり)
- 代替手段 1: マイナポータルから **CSV/PDF でダウンロードした薬剤情報を取り込む**
  インポート機能を作る (フォーマットが公開され次第 `server/src/routes/` にインポート
  ルートを追加するのが素直。QR 取り込みと同じく「解析 → プレビュー → 保存」の流れに乗せる)
- 代替手段 2: 電子処方箋の「処方箋 ID + 引換番号」の控えをメモとして記録する運用

## OS ヘルスアプリ連携

**現状: 未実装。実装するなら iOS を先行。**

- **iOS (HealthKit)**: iOS 16 以降、ヘルスケアアプリに服薬 (Medications) 機能があるが、
  サードパーティが読み書きできる **HKClinicalType / medication 系 API は米国の
  医療機関接続が前提**のものが中心。日本では歩数等と違い自由に書き込めない。
  Apple が medication の書き込み API を開放したら
  `app/src/health/` に health-assistant 方式 (HealthDataSource 抽象 + `native.ios.ts` 分岐) で追加する
- **Android (Health Connect)**: Health Connect に MedicationRecord 型が追加されるまで待ち
  (2026-07 時点でエクスペリメンタル)。追加されたら `react-native-health-connect` で対応
- 実装時の設計方針: 参考実装 health-assistant と同じく、ネイティブ SDK の import は
  `native.{ios,android,web}.ts` に隔離して Web バンドルに混ぜないこと

## 他アプリ連携

- **エクスポート**: JAHIS フォーマット (患者等⇒医療機関等、出力区分 2) のテキスト出力は
  パーサーの逆変換で実装できる。他のお薬手帳アプリへの移行手段として有用
- **共有 QR の生成**: 上記テキストを QR 画像にすれば薬局で見せられる
  (分割制御レコード 911 対応が必要。docs/jahis-qr-format.md 参照)
